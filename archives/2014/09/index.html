Setting up a Free Blog with Hexo and GitHub Pages

To start things off, I thought I’d give some insight into how I setup this blog using freely available software and hosting. I did pay for my domain name, but you don’t need to if you’re okay with hosting at username.github.io.

Hexo is “A fast, simple & powerful blog framework, powered by Node.js”. Honestly, I hadn’t heard of it before I started looking around for a freely hosted blog, but I liked its simplicity, Node.js-ness, and ability to customize.

Hexo is just blogging software, you’ll still need to host it somewhere. This is where GitHub Pages comes in. GitHub will host your website from a specially named GitHub repository for free. Pretty cool.

Here’s how I put these two things together to get this blog up and running.

Setup your GitHub Pages Repository

In order to get things started, you’ll need to create a GitHub account if you don’t have one already. They’re free, you can sign up at github.com.

Follow the first couple of steps from the GitHub Pages instructions to create and clone your repository. After you’ve cloned it, you don’t need to put anything in it just yet, we’ll take care of that using Hexo.

Download and Setup Hexo Locally

The Hexo Documentation provides installation instructions, and you should check them out if you run into any issues here. The key pre-requisite is that you’ll need to have Node.js and Git installed if you don’t already. Then, simply install Hexo with npm (Node Package Manager).

Once you have Hexo installed, change to the directory where you cloned your GitHub Pages repository and initialize your Hexo blog.

At this point, you should have an empty blog ready to go in here. Start the Hexo server

then visit http://localhost:4000 to check it out. You should see a mostly empty blog site with the default theme and a single, Hello World post.

Deploying your site to GitHub Pages

I’m going to skip over the details of writing posts and customizing your Hexo site for now, but check out the Hexo Documentation for more details on how to do this.

With the local site looking the way you want it to, you’ll want to publish it to GitHub pages for everyone to see. Hexo makes this super easy.

Firstly, edit the deploy configuration in the _config.yml file located in the root of your project.

Then, let Hexo do the heavy-lifting for the static site generation and deployment to GitHub Pages for you.

Done! You should now be able to see your site at //username.github.io. Note that it may take up to ten minutes the first time you visit the site for it to become available. This delay will only happen the first time while GitHub creates CNAME DNS records to route your user-specific domain name.
